C) CSS Files and Structure - top

Styles

We used 9 CSS files in this theme. We've merged styles for improving download speed template (for decreasing http queries).

Lets take a close look at our stylesheet files.

settings.css

This stylesheet contains Revolution slider styles .

gozha-nav.css

It contains styles for menu view. It has two formats of main menu view: a) full format (opened) menu – the menu that is used for large and medium displays (monitors, laptops, tablets (landscape orientation)); b) narrow format (compact) menu – styles that are used for small screens (phones, tablets (portrait orientation)). Transition point value is 768 pixels (is used as max-width condition for media query).

touch.css

Rules for touchscreen devices. The main purpose of this file is to provide certain hacks for mobile devices and to cancel animation used for desktop version.

style.css

Style.css is generated file of sets less files. It partly include 12-columns responsive grid system based on a default bootstrap 3.2.0 grid system & bootstrap styles and all custom style rules.

Less files structure:
  • base group:
    • normalize.less - a modern, HTML5-ready alternative to СSS resets;
    • vars.less - collection of custom variables (colors and fonts mostly);
    • base.less - custom rules for unclassed element. some most general rules. font init place;
    • ../external/bootstrap/less/bootstrap.less - grid system based on a default bootstrap 3.2.0 grid system & bootstrap styles for componetns;
  • object group:
    • objects.less - main file which contains styles for patterts;
  • state group:
    • state.less - contains rules for all patterns variations and responsive block.
    • responsive.less - contains responsive rules for all template (all media query rules).

The way to change color scheme.

If you want to change color scheme for Appic Template you can make it in two ways.

First way: choose one of existing color schemes. There are 18 stylesheets that already created if folder css (you can watch it live on appic website .) They have variation of light and dark background and different main colors (green, yellow, orange, emerald, blue and so on.). This is full list:

  1. style_blue.css
  2. style_blue_light.css
  3. style_default.css
  4. style_default_light.css
  5. style_emerald.css
  6. style_emerald_light.css
  7. style_green.css
  8. style_gren_light.css
  9. style_orange.css
  10. style_orange_light.css
  11. style_pink.css
  12. style_pink_light.css
  13. style_purple.css
  14. style_purple_light.css
  15. style_red.css
  16. style_red_light.css
  17. style_yellow.css
  18. style_yellow_light.css

Choose the suitable theme, replace default "style.css" file in your pages and enjoy it.

Second way: use less file and variables. First you should install program for less compile. One of posibible variant is Prepros (you can download this program for free, it works on Windows and Mac). Read a manual to this program. Than you should make several steps:

  • Add a template folder to working directoty.
  • Track this directory by compiled program.
  • Open file "vars.less" in folder "less/base".
  • Find 3 variables: @brand-primary, @brand-primary-darker and @brand-info.
  • Change @brand-primary amd @brand-primary-darker values for changing header and footer background and some primary color element in pages.
  • Change @brand-info and you will rewrite color for heading, buttons and some text on pages.
  • * Remember you can change only one variable, for example @brand-info only.
  • Save changes. Open browser and look at the results.

If you done all the steps correctly this manipulation will override your default "style.css" in folder css. So we advice you to make a backup first and you will have an opportunity to go back.

How to work with the main menu?

Main menu plugin initiates itself in file custom.js. It contains following options:

  • triggerMenu – tracks element if the code which performs switch role for mobile menu view.
  • subMenuTrigger – tracks elements serving for opening the dropdown menu in mobile view. (If your structure doesn’t provide dropdown menu this option is not necessary) .
  • AnimationSpeed – speed of hidden menu opening.

Twitter Feed

There are two steps to setup the Twitter Widget.

  1. Setup an application at Twitter dev center and get keys.
  2. Put your keys into Twitter files.
Setup Twitter application

  1. Visit https://dev.twitter.com/apps/ and sign in using your Twitter username and password. This doesn't have to be the username or password of the stream you need access to, just a Twitter account you control.
  2. Select 'Create new application' and enter the application details.
    1. The name and description can be anything you like really, but you can't use 'Twitter' in the name.
    2. The website field can be your main website and doesn't have to be the site where your Twitter feed or feeds are located.
    3. Callback URL can be left blank
  3. Enter the CAPTCHA info and click create
  4. On the next details screen, click 'create my access token'. You may need to refresh the page after a few seconds if it doesn't appear automatically.
  5. Make a note of the Consumer key, Consumer secret, Access token and Access token secret as highlighted below.
Setup Twitter application
  1. Open get-tweets.php file with help of your editor.
  2. Enter your twitter user name for $twitteruser = "";
  3. Enter your consumer key for $consumerkey = "";
  4. Enter your consumer secret key for $consumersecret = "";
  5. Enter your access token for $accesstoken = "";
  6. Enter your access token secret for $accesstokensecret = "";
  7. Open js/jquery.twitter.js file with help of your editor.
  8. Enter your twitter profile name for $twitterprofile = ""; (row 6)
  9. Enter your twitter screen name for $screenname = ""; (row 7)
